# План развития проекта SK New City

Проект представляет собой современное веб-приложение на Next.js 15, ориентированное на услуги по ремонту квартир. Архитектура чистая, с использованием Tailwind CSS, Radix UI и TypeScript.

## Текущее состояние
- **Frontend**: Next.js 15 (App Router), React 19, Tailwind CSS 4.
- **Контент**: Гибридный подход. Портфолио (properties) хранится в TS-файлах (`property-data.ts`), блоги — в Markdown (`markdown/blogs`).
- **SEO**: Настроена генерация метаданных и JSON-LD (Structured Data) для организации, услуг, FAQ и хлебных крошек.
- **UI/UX**: Использование анимаций (framer-motion не обнаружен, используются кастомные CSS-анимации и Intersection Observer), поддержка темной темы.

## Рекомендации по развитию

### 1. Оптимизация и расширение портфолио
- [ ] **Динамические макеты**: Сейчас в `Details` используется `switch` по `slug`. Рекомендуется перейти на более гибкую систему на основе метаданных в `property-data.ts`, чтобы избежать ручного добавления кейсов в код компонента.
- [ ] **Категории**: Исправить хардкод в `PropertiesListing`, где все категории маппятся на "Дизайнерский ремонт".
- [ ] **Фильтрация**: Добавить клиентскую фильтрацию по ЖК и площади на странице всех проектов.

### 2. Улучшение блога
- [ ] **MDX компоненты**: Использовать полноценный MDX для возможности вставки React-компонентов (например, калькуляторов или форм) прямо в статьи.
- [ ] **Пагинация**: Добавить пагинацию для списка блогов.

### 3. Технический долг и инфраструктура
- [ ] **Zod**: Внедрить валидацию данных для `property-data.ts` и контента из Markdown.
- [ ] **Image Optimization**: Проверить корректность `sizes` во всех компонентах для минимизации LCP.
- [ ] **Sentry/Error Tracking**: Добавить систему мониторинга ошибок.

### 4. Маркетинг и конверсия
